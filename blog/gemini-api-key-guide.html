在国内使用 Gemini API，主要面临两个问题：网络环境和支付门槛。目前主要有两种途径：​​自己申请（免费/付费）和购买第三方 API​​。自己申请虽然官方可靠，但有额度限制且需要网络支持；第三方则更省事，适合集成到应用中。

一、官方 API Key 获取（免费但有门槛）

这是最官方、成本最低的方式，但需要你准备好海外网络环境和一个谷歌账号。

申请步骤

  1. 访问 Google AI Studio (aistudio.google.com)。
  2. 登录你的谷歌账号（建议使用非主用账号，避免封号风险）。
  3. 在左侧边栏找到“API Keys”，点击右上角的“Create API Key”即可获得免费的 API 密钥。

免费额度与限制

  • 免费额度：官方提供免费层级，Gemini 1.5 Pro / Flash 等模型通常有每日请求限额（如 50-100 次/天，具体视政策而定）。
  • 核心限制：免费版有严格的 TPM (Tokens Per Minute) 和 RPM (Requests Per Minute) 限制。在处理长文档或高频调用时，很容易遇到 429 Too Many Requests 错误。
  • 数据隐私：免费版的输入数据可能会被 Google 用于改进模型（训练），因此不建议用于处理敏感数据。

四、国内直连解决方案

如果你已经有了官方 API Key，但在国内调用时遇到连接超时问题，可以使用以下两种方法实现直连。

1. 自建 Cloudflare Worker 反代 (技术方案)

利用 Cloudflare 免费的 Workers 服务，可以搭建一个专属的 API 转发通道。

worker.js 
export default {
  async fetch(request, env) {
    const url = new URL(request.url);
    // 将请求转发到 Google 的 API 域名
    url.hostname = 'generativelanguage.googleapis.com';
    
    const newRequest = new Request(url, {
      method: request.method,
      headers: request.headers,
      body: request.body
    });

    try {
      const response = await fetch(newRequest);
      return response;
    } catch (e) {
      return new Response(JSON.stringify({ error: e.message }), { status: 500 });
    }
  }
}

配置方法： 登录 Cloudflare -> Workers & Pages -> Create Application -> Create Worker，将上述代码粘贴进去并部署。你将获得一个 xxx.workers.dev 的域名，在代码中将 API Base URL 替换为该域名即可。

2. 使用 API 中转平台 (便捷方案)

如果你不想折腾代码，或者没有外币卡支付官方账单，选择正规的 API 中转平台是最优解。

  • 推荐理由：Gemini-VIP 等平台通常提供企业级并发支持，且价格往往优于个人直接购买（因为有企业折扣）。
  • 使用方式：购买额度包 -> 获取中转 API Key -> 将 Base URL 修改为中转商提供的地址（如 https://api.gemini-vip.top）。
  • 独享 Key：部分平台提供“独享 Key”服务，即帮你代申请官方 Key 并管理，既享受官方稳定性，又解决了支付和网络问题。

五、安全使用建议

API Key 安全

永远不要将 API Key 直接硬编码在客户端代码（如前端 JS、Android App）中，容易被抓包窃取。请务必在后端服务器发起调用。

成本控制

Gemini Pro 的 token 消耗很快。建议在开发阶段使用 Flash 模型（更便宜），生产环境再切换到 Pro。同时设置 Usage Limit（用量限制），防止意外产生高额账单。
